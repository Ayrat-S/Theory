1) 53 Test plan. - 32стр. ???
*A document describing the scope, approach, resources and schedule of intended test activities. It identifies amongst
others test items, the features to be tested, the testing tasks, who will do each task, degree of tester independence, the test
environment, the test design techniques and entry and exit criteria to be used, and the rationale for their choice, and any risks
requiring contingency planning. It is a record of the test planning process. [ISTQB Glossary]
*Документ, описывающий объем, подход, ресурсы и график предполагаемых тестовых действий. Он определяет среди
другие элементы тестирования, тестируемые функции, задачи тестирования, кто будет выполнять каждую задачу, степень независимости тестировщика, тест
среду, используемые методы разработки тестов и критерии входа и выхода, а также обоснование их выбора и любые риски.
требующие планирования на случай непредвиденных обстоятельств. Это запись процесса планирования тестирования. [Глоссарий ISTQB]

2) 146 Negative testing. - 79стр. Негативное тестирование
*Tests aimed at showing that a component or system does not work. Negative testing is related to the testers’
attitude rather than a specific test approach or test design technique, e.g. testing with invalid input values or exceptions. [ISTQB
Glossary]
*Тесты, направленные на то, чтобы показать, что компонент или система не работает. Отрицательное тестирование связано с
отношение, а не конкретный подход к тестированию или метод разработки тестов, например. тестирование с недопустимыми входными значениями или исключениями. [ISTQB
Глоссарий]

3) 153 Exploratory testing. - 82стр. Исследовательское тестирование
An informal test design technique where the tester actively controls the design of the tests as those tests
are performed and uses information gained while testing to design new and better tests. [ISTQB Glossary]

4) 155 Checklist-based Testing - 82стр ???
An experience-based test design technique whereby the experienced tester uses a high-level list of
items to be noted, checked, or remembered, or a set of rules or criteria against which a product has to be verified. [ISTQB
Glossary]

5) 157 Ad hoc testing - 82стр ???
Testing carried out informally; no formal test preparation takes place, no recognized test design technique is
used, there are no expectations for results and arbitrariness guides the test execution activity. [ISTQB Glossary]

6) 223 Error Guessing. - 91стр ???
A test design technique where the experience of the tester is used to anticipate what defects might be present
in the component or system under test as a result of errors made, and to design tests specifically to expose them. [ISTQB
Glossary]

7) 228 Equivalence partitioning - 91стр. ???
A black box test design technique in which test cases are designed to execute representatives from
equivalence partitions. In principle test cases are designed to cover each partition at least once. [ISTQB Glossary]

8) 229 Boundary value analysis. - 92стр Анализ граничных значений
A black box test design technique in which test cases are designed based on boundary values (input
values or output values which are on the edge of an equivalence partition or at the smallest incremental distance on either side
of an edge, for example the minimum or maximum value of a range). [ISTQB Glossary]

9) 230 Domain analysis. - 92стр. ???
A black box test design technique that is used to identify efficient and effective test cases when multiple variables can or should be tested together. It builds on and generalizes equivalence partitioning and boundary values analysis. [ISTQB
Glossary]

10) 231 Pairwise testing. - 92стр Попарное тестирование
A black box test design technique in which test cases are designed to execute all possible discrete combinations
of each pair of input parameters. [ISTQB Glossary]

11) 232 N-wise testing - 92стр ???
A black box test design technique in which test cases are designed to execute all possible discrete combinations
of any set of n input parameters. [ISTQB Glossary]

12) 239 Data Flow Testing - 93стр
A white box test design technique in which test cases are designed to execute definition-use pairs of variables.
[ISTQB Glossary]

13) 240 State Transition Testing. - 94стр 
A black box test design technique in which test cases are designed to execute valid and invalid state
transitions. [ISTQB Glossary]

14) 246 Statement Testing. - 94стр
A white box test design technique in which test cases are designed to execute statements (statement is an
entity in a programming language, which is typically the smallest indivisible unit of execution). [ISTQB Glossary]

15) 247 Branch Testing. - 94стр
A white box test design technique in which test cases are designed to execute branches (branch is a basic block
that can be selected for execution based on a program construct in which one of two or more alternative program paths is
available, e.g. case, jump, go to, if-then-else.). [ISTQB Glossary]


16) 248 Condition Testing. - 95стр
A white box test design technique in which test cases are designed to execute condition outcomes (condition
is a logical expression that can be evaluated as True or False, e.g. A > B). [ISTQB Glossary]

17) 249 Multiple Condition Testing. - 95стр
A white box test design technique in which test cases are designed to execute combinations of single
condition outcomes (within one statement). [ISTQB Glossary]

18) 251 Decision Testing. - 95стр
A white box test design technique in which test cases are designed to execute decision outcomes (decision is
program point at which the control flow has two or more alternative routes, e.g. a node with two or more links to separate
branches). [ISTQB Glossary]

19) 252 Path testing. - 95стр
A white box test design technique in which test cases are designed to execute paths. [ISTQB Glossary]

20) 254 Decision Table Testing - 96стр
A black box test design technique in which test cases are designed to execute the combinations of
inputs and/or stimuli (causes) shown in a decision table (a table showing combinations of inputs and/or stimuli (causes) with their
associated outputs and/or actions (effects), which can be used to design test cases). [ISTQB Glossary]

21) 256 Use case testing. - 96стр
A black box test design technique in which test cases are designed to execute scenarios of use cases. [ISTQB
Glossary]

22) 257 User story testing. - 97стр
A black box test design technique in which test cases are designed based on user stories to verify their correct
implementation. [ISTQB Glossary]

23) 259 Random testing. - 97стр
A black box test design technique where test cases are selected, possibly using a pseudo-random generation
algorithm, to match an operational profile. This technique can be used for testing non-functional attributes such as reliability and
performance. [ISTQB Glossary]

24) 267 Classification tree method. - 104стр
A black box test design technique in which test cases, described by means of a classification tree,
are designed to execute combinations of representatives of input and/or output domains. [ISTQB Glossary]

25) 268 Syntax testing - 104стр
A black box test design technique in which test cases are designed based upon the definition of the input domain
and/or output domain. [ISTQB Glossary]

26) 273 Cause-effect graphing. - 104стр
A black box test design technique in which test cases are designed from cause-effect graphs (a graphical
representation of inputs and/or stimuli (causes) with their associated outputs (effects), which can be used to design test cases).
[ISTQB Glossary]

27) 274 Data flow testing. - 105стр
A white box test design technique in which test cases are designed to execute definition-use pairs of variables.
[ISTQB Glossary]

28) 327 Test plan. - 208стр
A document describing the scope, approach, resources and schedule of intended test activities. It identifies amongst
others test items, the features to be tested, the testing tasks, who will do each task, degree of tester independence, the test
environment, the test design techniques and entry and exit criteria to be used, and the rationale for their choice, and any risks
requiring contingency planning. It is a record of the test planning process. [ISTQB Glossary]

29) 329 Test approach. - 209стр
The implementation of the test strategy for a specific project. It typically includes the decisions made that follow
based on the (test) project’s goal and the risk assessment carried out, starting points regarding the test process, the test design
techniques to be applied, exit criteria and test types to be performed. [ISTQB Glossary]
