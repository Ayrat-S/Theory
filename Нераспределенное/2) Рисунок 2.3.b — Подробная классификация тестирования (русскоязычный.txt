
Рисунок 2.3.b — Подробная классификация тестирования.

Ниже описана иерархическая структура рисунка:

- По запуску кода на исполнение
	- Статическое тестирование
	- Динамическое тестирование

- По доступу к коду и архитектуре приложения
	- Метод белого ящика 
	- Метод черного ящика
	- Метод серого ящика

- По степени автоматизации
	- Ручное
	- Автоматизированное (и "автоматическое")

- По уровню детализации приложения (по уровню тестирования)
	- Модульное тестирование ("смоук")
	- Интеграционное тестирование
	- Системное тестирование

- По (убыванию) степени важности тестируемых функций (по уровню функционального тестирования)
	- Дымовое тестирование
	- Критического пути
	- Расширенное

- По природе приложения
	- Веб
	- Мобильное
	- Настольное
	- ...

- По фокусировке на уровне архитектуры приложения
	- Уровень представления
	- Уровень бизнес логики
	- Уровень данных

- По привлечению конечных пользователей
	- Альфа тестирование
	- Бета тетсирование
	- Гамма тестирование

- Ппо степени формализации
	- На основе тест-кейсов
	- Исследовательское
	- Свободное (интуитивное)

- По техникам и подходам
	- Позитивное
	- Негатвное
	- На основе (знания) источников ошибок
		- Предугадывание ошибок
		- Эвристическая оценка
		- Добавление ошибок
		- Мутационное
	- На основе опыта тестировщика, сценариев, чек-листов
		- Исследовательское
		- Свободное (интуитивное)
	- По степени вмешательства в работу приложения
		- Инвазивное
		- Неинвазивное
	- По техникам автоматизации
		- Под управлением данными
		- Под управлением ключевыми словами
		- Под управлением поведением
	- На основе среды выполнения
		- Тестирование в процессе разработки
		- Оперцаионное тестирование
	- На основе выбора входных данных
		- На основе классов эквивалентности
		- На основе граничных условий
		- Доменное
		- Попарное
		- На основе ортогональных массивов
	- На основе (моделей) поведения приложения
		- По табице принятия решений
		- По диаграмме или таблице состояний
		- По спецификациям
		- По моделям поведения приложения
		- На основе вариантов использования
		- Параллельное
		- На основе случайных данных
		- A / B
	- На основе структур кода
		- Выражений
		- Ветвей
		- Условий
		- Комбинаций условий
		- Решений
		- Решающих условий
		- Путей
	- На основе кода
		- По потоку управления
		- По потоку данных
		- По диаграмме или таблице состояний
		- Инспекция (аудит) кода

- По целям и задачам
	- По принципам работы с приложением
		- Позитивное
		- Негативное
	- Функциональное
	- Нефункциональное
	- Инсталляционное
	- Регрессионное
	- Повтороное
	- Приемочное
	- Удобства использования
	- Доступности
	- Интерфейса
	- Безопасности
	- Интернационализации
	- Локализации
	- Совместимости
